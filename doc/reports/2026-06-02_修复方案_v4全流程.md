## 1. 参考文献顺序修复 [🔴 优先级最高]

### 当前问题

`src/writing/revision_engine.py:312-412` — 三段式逻辑（分离循环 + 修订循环 + 组装循环）导致 refs 位置每轮上移。

### 修复

将三段式改为：先跑修订产出 `revised_map`（串行/并行分支不动），最后单次遍历 `manuscript.sections` 组装 `new_sections`。

**文件**：`src/writing/revision_engine.py`

**改动位置**：`revise_manuscript()` 中 L312-L412 整个构建 `new_sections` 的逻辑块。

**改动思路**：

```
修改前（三段式，有错位风险）：
  step1: 分离循环 — refs 直接 append，其余推迟
  step2: 修订循环 — 产出 revised_map
  step3: 组装循环 — 其余章节追加在后面

修改后（单次遍历组装）：
  先跑修订逻辑产出 revised_map（step2 不变）
  最后单次遍历 manuscript.sections，统一处理四种情况：
    refs      → append 重建的 refs
    keywords  → skip
    在 revised_map 中 → append 修订过的版本
    否则（不在 section_map 中） → append 原样保留
```

**同时顺手处理 `reference_before_acknowledgment` 配置**：当前该配置只在 `draft_manuscript` 组装阶段（`draft_engine.py:611-618`）生效，修订后的重建不尊重这个配置。单次遍历时加条件判断：如果配置为 true，refs 应在 ack 之前；默认 false 则 refs 在最后。

```python
# 单次遍历组装伪代码
refs_before = bool(get("reference_before_acknowledgment", False))
refs_sec = _build_ref_list_section(store, thesis_mode)
found_ack = False

for sec in manuscript.sections:
    if sec.section_id == "refs":
        if not refs_before:
            continue  # 跳过，等遍历完再放最后
        new_sections.append(refs_sec)
    elif sec.section_id == "acknowledgment":
        new_sections.append(sec)
        if not found_ack and not refs_before:
            new_sections.append(refs_sec)  # 默认：refs 放在致谢之后（即最后）
        found_ack = True
    elif sec.section_id == "keywords":
        continue
    elif sec.section_id in revised_map:
        new_sections.append(revised_map[sec.section_id])
    elif sec.section_id in section_map:
        new_sections.append(sec)
    else:
        new_sections.append(sec)

if not found_ack:
    if refs_before:
        new_sections.append(refs_sec)
    new_sections.append(
        ManuscriptSection(section_id="acknowledgment", title="致谢", markdown_body="...")
    )
    if not refs_before:
        new_sections.append(refs_sec)
```

### 改动量

删除 ~20 行三段式 + 追加 ~15 行单循环 = 净减约 5 行。不触动修订 prompt / LLM 调用 / 并行线程等核心逻辑。

## 3. 英文摘要严格直译 [🟡]

### System Prompt 重写

**文件**：`src/writing/abstract.py`  
**位置**：`_SYSTEM_EN_ABSTRACT_FROM_ZH` 常量（约 L58-80）

### 问题分析

现行 System Prompt 包含 `polished academic English`、`tight and information-dense`、`correct any unidiomatic constructions` 等措辞，授予 LLM 过大的压缩/改写权限。

### 修改后 System Prompt

```python
_SYSTEM_EN_ABSTRACT_FROM_ZH = """# Role
You are a strict academic translator (Chinese → English). Your only job is to produce
an English rendering that preserves every fact, structure, term, and nuance of the
Chinese source abstract.

# Task
Translate the provided Chinese abstract into English sentence by sentence.

# Constraints
1. DO NOT summarize, compress, reorganize, or embellish — preserve ALL information
2. Each Chinese sentence must correspond to one English sentence (one-to-one mapping)
3. Technical terms must match the Chinese source exactly (do not expand acronyms)
4. Terminology: use "this thesis" (not "paper" or "article"); keep acronyms as-is
5. Voice: passive preferred; no "I", "we", "our"
6. Tense: past for methods/results; present for conclusions
7. Output ONLY the English abstract body — no heading, no commentary

# Strictly Forbidden
- Shortening a 5-paragraph abstract into 2 paragraphs
- Deleting technical details to "be more concise"
- Reordering the four elements (Objective → Methods → Results → Conclusion)
- Adding commentary or evaluation not present in the source
""" + _EXECUTION_PROTOCOL  # ← 保持与原 prompt 一致的执行约束
```

### 改动量

~20 行替换，末尾保留 `_EXECUTION_PROTOCOL` 拼接，不改调用逻辑。

### 4D — 小节主题对齐强约束

#### D1：`_build_draft_prompt` 追加对齐声明

**文件**：`src/writing/draft_engine.py`  
**位置**：`_build_draft_prompt()` 中「强制小节结构」块之后

**改动**：追加约 12 行：

```python
if section.subsections:
    ch_num = int(section.section_id[1:]) if section.section_id[1:].isdigit() else 0
    chapter_topic = section.title
    if ch_num:
        chapter_topic = chapter_topic.replace(f"第{ch_num}章 ", "").strip()
    align_lines = [
        "\n【小节-章节主题对齐（强制）】",
        "以下每个小节必须直接服务于本章总主题：",
    ]
    for sub in section.subsections:
        align_lines.append(
            f"- {sub.title} → 该小节的第一句话必须说明本部分"
            f"在「{chapter_topic}」中的角色与必要性"
        )
    align_lines.extend([
        "禁止将独立成章的内容降级为一个小节（如'通信环境建模'不应作为'模型设计'章的一节）",
        "若某个小节内容已构成独立主题，应重新审视其章节归属",
    ])
    parts.append("\n".join(align_lines))
```
