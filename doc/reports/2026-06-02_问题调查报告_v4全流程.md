## 问题 1：参考文献未放在最后 [🔴 Bug]

### 现象

输出文件 `## 参考文献`（第 67 行）出现在 `## 摘要`（第 100 行）和正文之前。

### 根因

`src/writing/revision_engine.py:312-412` 的 `revise_manuscript` 用**三段式**逻辑构建 `new_sections`，导致每次修订后参考文献位置上移：

1. **分离循环**（L312-L322）：遍历 `manuscript.sections`，refs 直接 append 到 `new_sections`，其余章节收集进 `revise_sections`
2. **修订循环**（L325-L407）：并行/串行修订产出 `revised_map`
3. **组装循环**（L410-L412）：遍历 `manuscript.sections`，把 `revised_map` 里的结果 append 到 `new_sections`

**问题**：步骤 1 中 refs 在原位被 append 到 `new_sections`，步骤 3 中其余章节在后面追加。如果初始顺序是 `[abstract, s1, s2, ..., refs]`，第一轮修订后变成 `[refs, abstract, s1, ..., ack]`——refs 跑到了最前面。第二轮修订时，manuscript.sections 已是 `[refs, abstract, s1, ..., ack]`，分离循环再次将 refs（第一个元素）提前 append，导致 refs 在不同轮次间持续前移。（注：refs 不会重复——L313 的 `continue` 阻止 refs 进入 `revise_sections`，第二轮组装循环中 refs 不在 `revised_map` 中，不会被二次 append。）

具体过程：

```
第一轮：manuscript.sections = [abstract, s1, s2, ..., refs, ack]
  step1: refs append → new_sections=[refs]
  step3: s1..s6 ack append → new_sections=[refs, abstract, s1, ..., ack]

第二轮：manuscript.sections = [refs, abstract, s1, ..., ack]  ← 来自上一轮
  step1: refs append (仍是第一个) → new_sections=[refs]
  step3: 其余 append → new_sections=[refs, abstract, s1, ..., ack]
```

经过三轮修订（v4），refs 被推到摘要之前。

### 代码证据

```python
# revision_engine.py L312-L322（分离循环 — refs 直接塞入，其余推迟）
for sec in manuscript.sections:
    if sec.section_id == "refs":
        new_sections.append(_build_ref_list_section(store, thesis_mode))
        continue                                       # ← 不进入 revise_sections
    if sec.section_id == "keywords":
        continue
    ...
    revise_sections.append(sec)                        # ← 推迟到修订

# L325-L407：修订循环 → 产出 revised_map
# L410-L412（组装循环 — 其余章节跟在 refs 后面）
for sec in manuscript.sections:
    if sec.section_id in revised_map:
        new_sections.append(revised_map[sec.section_id])
```

## 问题 3：英文摘要不是中文摘要的直译 [🟡 设计偏差]

### 现象

英文摘要（第 110 行）内容结构与中文摘要（第 101-106 行）存在显著差异——信息更少、结构不同。

### 根因

`src/writing/abstract.py:58-80` — `_SYSTEM_EN_ABSTRACT_FROM_ZH` 赋予了 LLM"学术编辑"自由润色的权限：

```
You are a bilingual academic editor... Translate the provided Chinese
abstract into polished academic English
```

关键 token：`polished`、`tight and information-dense`（暗示压缩）、`correct any unidiomatic constructions`（暗示改写）。LLM 将"润色权"理解为可以压缩/重组/选择性翻译。

### 冲突分析

| 层级 | 措辞 | 语义 |
|------|------|------|
| user prompt | `Please translate the following...` | 严格翻译 |
| system prompt | `polished academic English`、`tight` | 自由润色 |
结果：LLM 倾向于遵从 System Prompt。

## 问题 4：内容一致性不足 [🟡 系统级]

### 现象

评估报告指出「第3章标题为'基于DQN的功率控制模型设计'，但3.1节'V2X通信环境建模'以信道模型和仿真环境搭建为主，偏离模型设计主题」。

### 根因分析

**当前防御层次及其有效性**：

| 层次 | 位置 | 机制 | 实际效果 |
|------|------|------|----------|
| `scope_must_include` | `draft_engine.py` | 正则子串检查 | ⚠️ 只能验证"是否包含"，无法验证"是否越界" |
| `_SECTION_RULES` | `helpers.py` | Prompt 级建议 | ⚠️ LLM 频繁忽略 |
| `_SYSTEM_DRAFT_THESIS` | `draft_engine.py` | 无章-小节一致性规则 | ❌ 缺失 |
| 评估器 | `evaluator.py` | 事后标注 | ⚠️ 只能发现，无法预防 |

**根本矛盾**：所有约束都是软 prompt，无确定性机制保证小节归属正确。LLM 在"3.1 V2X通信环境建模"标题下天然倾向写通信环境背景——这在语义上合理，但在论文结构中放错了位置。
